Features

Our project aims to develop a tool for financial risk management. More precisely, it provides three different methods to compute the Value-at-Risk, a measure of the risk of loss on portfolio of financial assets. Our software assists the user through his decision-making process, from the creation of portfolio to the comparison of the three methods:
  1. Historic method: the simplest method
  2. GARCH modelisation: based on a volatility and returns model
  3. Riskmetrics method: heuristic-based method using a particular case of the GARCH modelisation
In its first stage, our software will allow the user to produce several reports as PDF or DOCX documents.
Currently, it is the only open-source product on the market to provide such services. The software is licenced under the LGPL 3.

How does it work?

Our software core is entirely written in C++. However, it uses several other languages and frameworks for certain features.
It relies on R, which is a free software environnement for statistical computing. R is used for several algorithms such as the GARCH estimation and the Riskmetric algorithm.
Our graphical user interface uses the Qt widget toolkit. It also creates the charts which are then included in the reports.
To generate the reports, we developed a light Java command-line tool. It uses the open-source API xdocreport and receives the values to include in the report as JSON on the standard input.

Continuous integration

Our projet is hosted on GitHub. Sources and executables can be downloaded there.
We used Open-source software development methods throughout the project. All new features or improvements are developped on new branches and then merged into the project using Pull Requests. The branch is entirely reviewed by another contributor before merging. This method allows us to produce a far better quality software.
